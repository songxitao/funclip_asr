## 0. 方案演进与背景

### 0.1 盲滑窗方案（上一阶段）的致命局限
上一阶段实现的“盲滑窗”（`diarize_strategy == "sliding"`）方案虽然将 CONF（说话人混淆）从 48.3% 降至 26.3%，但引入了以下工程缺陷：
1. **静音区 None 填充致使 FA 虚高**：在整段音频上盲目滑窗，静音和背景噪段由于声能不足导致 Cam++ 提取 Embedding 失败返回 `None`。原实现使用前向有效标签填充，导致静音区被强行标记为有人说话，FA 从 0.3% 升至 3.4%。
2. **话序打乱与字幕合并冲突**：滑窗切碎了时间线，而 ASR 转写是基于 VAD 粗段（~8s）输出的。若在 ASR 段内发生了说话人切换，在没有词级时间戳（Word-level timestamp）的情况下，无法完美将这句文本拆分给不同的说话人。若强行按字数比例切分且声纹聚类有抖动，会导致转写字幕的话序和语义严重错乱。
3. **边界精度退化**：滑窗合并后的起止时间受限于 0.5s 的步长，存在截断误差。

### 0.2 核心策略：VAD 活性段 + 段内滑窗提纯（VAD-Constrained Sliding Mean）
为了保住转写文字的语义连贯底线，且解决 VAD 大段声纹容易被段内多人污染的问题，本次方案做出以下设计决策：
*   **以 VAD 活性大段为基本输出单位**：ASR 识别结果与说话人分配完全以 VAD 段为基准（1 个 VAD 段对应 1 句文本与 1 个 Speaker ID）。这保证了**绝不搞乱字幕次序**。
*   **段内滑窗提纯（Sliding Mean）**：在提取某个 VAD 段的声纹 Embedding 时，不再对这 8s 整段提取单个向量，而是在该段内部进行 1.5s 窗 + 0.5s 步长的滑窗提取，对过滤后的有效子向量计算 **算术平均（Mean Embedding）**，以此代表该 VAD 段参与全局聚类。
*   **静音天然过滤**：由于滑窗完全限制在 VAD 语音活性段内，彻底消除了静音区提声纹失败的问题，FA 降回 0.3% 的极佳水平。

## 1. 核心设计与数据流

新方案的数据流与模块分工如下：

```
                      ┌──────────────────────┐
                      │    整段 16k 音频      │
                      └──────────┬───────────┘
                                 │
                      ┌──────────▼───────────┐
                      │  ASR VAD (FSMN-VAD)  │
                      └──────────┬───────────┘
                                 │
                      ┌──────────▼───────────┐
                      │    VAD 活性大段列表   │
                      │  (等价于 ASR chunk)  │
                      └──────────┬───────────┘
                                 │
         ┌───────────────────────┴───────────────────────┐
         ▼ (ASR 分支)                                    ▼ (说话人分离分支)
   ┌───────────┐                                 ┌────────────────────────┐
   │ ASR 模型  │                                 │ 针对每个 VAD 大段：     │
   │ 识别文本  │                                 │ 在大段内部以 1.5s 窗、 │
   │           │                                 │ 0.5s 步长滑动切片       │
   └─────┬─────┘                                 └───────────┬────────────┘
         │                                                   │
         │                                       ┌───────────▼────────────┐
         │                                       │ 提取各窗口 Embedding，  │
         │                                       │ 剔除 None 后进行均值聚合│
         │                                       │ -> 段级 Mean Embedding  │
         │                                       └───────────┬────────────┘
         │                                                   │
         │                                       ┌───────────▼────────────┐
         │                                       │ 对所有段的 Mean Emb    │
         │                                       │ 统一进行全局谱聚类      │
         │                                       └───────────┬────────────┘
         │                                                   │
         └───────────────────────┬───────────────────────────┘
                                 │ (一一对齐合并)
                      ┌──────────▼───────────┐
                      │ [说话人i] ASR文本    │
                      │ (首尾完全对齐VAD边界) │
                      └──────────────────────┘
```

### 1.1 均值聚合算法（VAD-Constrained Sliding Mean）
对于第 $i$ 个 VAD 段 $C_i$（长度为 $L_i$ 采样点）：
1. 若 $L_i < 1.5 \text{s} \times 16000$，不进行滑窗，直接提取该段整体的声纹向量 $E_i = \text{Extract}(C_i)$。
2. 若 $L_i \ge 1.5 \text{s} \times 16000$，则在段内以 1.5s 窗长（24000 采样点）和 0.5s 步长（8000 采样点）滑动切片，得到子窗口列表 $w_{i, j}$。
3. 逐个子窗口提取声纹：$e_{i, j} = \text{Extract}(w_{i, j})$。
4. 收集所有非 `None` 的声纹向量，进行算术平均归一化：
   $$E_i = \text{Normalize}\left( \frac{1}{N} \sum_{j=1}^{N} e_{i, j} \right)$$
   其中 $\text{Normalize}$ 为 $L_2$ 归一化。若所有子窗均为 `None`，则退化为对 $C_i$ 整体提取。
5. 将提纯后的 $E_i$ 作为段 $C_i$ 的最终声纹表征，放入全局 Embedding 矩阵中参与聚类。

## 2. 改造范围与实现路径

我们将在现有的 `diarize_strategy` 链条中新增或重构一个策略 `vad_sliding`：

### 2.1 speaker_engine.py
*   **方法重构**：在 `CampPlusSpeaker` 类中新增段级滑窗提纯方法 `extract_embedding_sliding_mean(self, chunk_16k, sr=16000)`。
*   **聚类分发**：`cluster(..., strategy="vad_sliding", ...)`：
    - 遍历传入的 `chunks`。
    - 对每个 `chunk` 调用 `extract_embedding_sliding_mean()` 获得提纯向量。
    - 将聚合后的矩阵送入现有的谱聚类（Spectral）或层次聚类（AHC）。

### 2.2 asr_onnx_service.py
*   **合并逻辑**：
    - `/transcribe` 的 `diarize_strategy` 参数新增 `"vad_sliding"`（设为默认值，取代旧的 `"two_stage"`）。
    - 走 `diarize_strategy == "vad_sliding"` 分支时，直接调用 `cluster(chunks, strategy="vad_sliding")`  或者直接映射。
    - 将聚类返回的段级 `speaker_id` 映射到对应的 `clean_texts[i]`。
    - 恢复输出 `segments` 中的 `text` 字段回填，使 `diarized_text` 不再为空。

### 2.3 der_eval.py 与评测
*   **支持新策略**：在评测脚本中加入 `--diarize_strategy vad_sliding`。
*   **评测指标验证**：在 AliMeeting near 数据集上重新跑全量 20 场，对比 `vad_sliding` 相比于原有 `two_stage` 方案的性能收益，重点验证：
    1. CONF 是否由于滑窗提纯而显著下降。
    2. FA 是否如预期重新回落到 0.3% 左右的极佳水平。
    3. 字幕时序与语义连贯性是否完全不受影响。

## 3. 待验证与自检 (Spec Self-Review)

*   **1.5s 窗在短 VAD 段中的覆盖率**：当 VAD 段本身只有 2.0s 时，滑窗切片会产生 2 个高重叠窗，求平均效果是平滑的。
*   **Cam++ 在多段平均下的区分度**：算术平均对于声纹提取器特征空间的线性保留度已经在业界（如 Pyannote 对 VAD 区域池化）被广泛验证可行。
*   **None 兜底机制**：如果某段里所有的滑窗均因为音量太低提取失败，代码需要优雅降级为提取整段，若依然失败，则全局分配为默认说话人，防止服务崩塌。
